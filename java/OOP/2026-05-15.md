# 인터페이스, 추상 클래스, 상속
오늘은 추상 클래스에 대해 설명하겠다.                                   
보통 우리가 생각하는 클래스는 직접 객체를 생성할 수도 있고, 상속도 가능하다. 하지만 이것은 추상 클래스
추상 클래스는 직접 객체를 생성할 수 없으며 상속을 통해 자식 클래스에게만 떠넘긴다. 
이러한 방식으로 인해 abstract라는 키워드를 사용한다. 이러면 다형성을 쓸 수 있기 때문이다.

이래서 추상 클래스는 거의 무조건 오버라이딩을 써야한다. 오버라이딩은 부모한테 받은 메서드를
자기 방식대로 그냥 뜯어고치는 것이다.

```java
// 추상 클래스 정의
abstract class Shape {
    String color;

    // 추상 메서드 (구현부 없음)
    abstract void draw();

    // 일반 메서드
    void setColor(String color) {
        this.color = color;
    }
}

// 추상 클래스를 상속받는 자식 클래스
class Circle extends Shape {
    // 필수: 추상 메서드 오버라이딩
    @Override
    void draw() {
        System.out.println("원을 그립니다.");
    }
}
```
이렇게 할 수 있다.
