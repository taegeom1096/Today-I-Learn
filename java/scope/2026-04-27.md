# 명시적 형변환
반대로 큰 범위에서 작은 범위로 대입하려면 명시적 형변환이 필요하다.

예를 들면 double형은 실수로 표현할 수 있는데 int는 실수를 표현할 수 없다. 
이에 따라 double -> int를 대입하면 컴파일 오류가 발생한다. 
하지만 double형 변수 앞에 (int)를 붙여주면 int형으로 바뀌어 1.5의 값을 1로 출력하게 된다.
하지만 이것은 데이터 타입을 강제로 바꾸는 것이기에 개발자가 위험을 감수해야 한다.

## 오버 플로우
오버 플로우는 자료형의 범위값이 벗어나면 생긴다.
얘를 들어 int는 -2147483648 ~ 2147483647 정도 되는데 이 범위를 벗어나면 오버 플로우가 발생한다
오버 플로우를 얘방하는 방법은 그냥 더 큰 자료형을 쓰면 된다.
int a = 2147483648이라고 지정했으면 
(doube) a = 2147483648L이라고 다시 형변환하고 출력하면 오버 플로우가 발생하지 않는 것을 볼 수 있다.
