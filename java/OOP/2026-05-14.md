# 인터페이스, 추상클래스, 상속

오늘은 인터페이스에 대해 **자세히** 정리하겠다.

First 인터페이스에 대해 한번 explain 해보겠다.
인터페이스란? 클래스들이 구현해야 하는 공통 규격을 정의하는 추상 타입이다.
한편 인터페이스는 클래스랑 비슷한데 왜 인터페이스를 쓰는가? 한 번 알아보기 전 이해하기 쉽게 먼저 예시코드르 보자

```java
// 설계도 작성
interface Animal {
    void sound(); // 추상 메소드
}

// 설계도 구현
class Dog implements Animal {
    public void sound() {
        System.out.println("멍멍");
    }
}

class Cat implements Animal {
    public void sound() {
        System.out.println("야옹");
    }
}
```
위와 같은 코드를 설명하면 Animal이라는 추상 메소드에 sound()를 정의하고 거기서 Cat과 Dog가 그걸 끌어다 쓰는 원리이다.

그런데 class Animal을 왜 쓰지 않았을까? 그 이유는                                          
자바에서 클래스는 단 하나의 부모 클래스만 상속할 수 있다. 인터페이스는 개수 제한 없이 여러 개를 동시에 구현할 수 있다.
이러한 점으로 인터페이스는 내용 없는 껍데기만 사용할 때 많이 사용된다.
